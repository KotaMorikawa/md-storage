# Markdown Saver - ブラウザ拡張機能

Web ページを Markdown 形式で保存する Chrome/Firefox 拡張機能です。シームレスなディレクトリ選択機能により、指定したフォルダに直接保存できます。

## ✨ 主要機能

### 📝 コンテンツ変換

- **高精度な Markdown 変換**: Turndown.js による高品質な HTML→Markdown 変換
- **メタデータ自動付与**: 保存日時、元 URL、タイトルを自動記録
- **画像・リンク処理**: 相対 URL の絶対 URL 変換、alt 属性保持
- **テーブル対応**: HTML テーブルを Markdown 表形式に変換
- **不要要素除去**: 広告、ナビゲーション、SNS ボタンの自動削除

### 🗂️ 保存機能

- **シームレスディレクトリ保存**: 一度選択すれば権限が保持される
- **自動ファイル名生成**: ページタイトルから安全なファイル名を生成
- **重複回避**: 同名ファイル存在時の自動採番（`document_1.md`, `document_2.md`）
- **フォールバック保存**: File System API 非対応時はダウンロードフォルダに自動保存
- **権限管理**: ディレクトリアクセス権限の自動監視・復元

### 🔒 セキュリティ & プライバシー

- **最小権限の原則**: 選択されたディレクトリのみにアクセス
- **権限の透明性**: リアルタイムな権限状態表示
- **自動権限クリア**: エラー時の権限自動削除
- **HTTPS 必須**: 暗号化通信でのみファイルシステムアクセス可能

## 🔐 セキュリティについて

### File System Access API の安全性

**✅ 厳格な制限**

- **選択ディレクトリのみ**: ユーザーが明示的に選択したフォルダのみアクセス可能
- **システムファイル保護**: OS 重要ディレクトリは選択不可（`C:\Windows`, `/sys`, `/dev` など）
- **Same-Origin 制限**: 権限は付与された Web サイトのみで有効
- **明示的同意**: すべてのアクセスはユーザーの承認が必要

**🛡️ 多層防御**

- **HTTPS 強制**: 暗号化通信でのみ動作
- **ユーザーアクション必須**: クリックなどの明示的操作が必要
- **権限監視**: 定期的な権限状態確認と自動クリア
- **読み取り専用動作**: 既存ファイルの変更・削除は行わない

**⚠️ 推奨事項**

- 専用フォルダの作成を推奨（例: `Documents/MarkdownSaves/`）
- 重要な文書と同じフォルダの使用は避ける
- ブラウザ設定で権限を定期的に確認

### データプライバシー

- **ローカル処理**: すべての変換処理はブラウザ内で完結
- **外部送信なし**: ページコンテンツは外部サーバーに送信されません
- **最小限のストレージ**: ディレクトリ名と権限状態のみをローカル保存

## 🚀 インストール方法

### Chrome（開発者モード）

1. Chrome を開き、`chrome://extensions/` にアクセス
2. 右上の「開発者モード」を有効にする
3. 「パッケージ化されていない拡張機能を読み込む」をクリック
4. このプロジェクトのフォルダ（`md-storage`）を選択

### Firefox（一時的なアドオン）

1. Firefox を開き、`about:debugging` にアクセス
2. 「この Firefox」をクリック
3. 「一時的なアドオンを読み込む」をクリック
4. `manifest.json` ファイルを選択

### Brave ブラウザでの利用

Brave はプライバシーのため File System Access API をデフォルト無効にしています：

1. `brave://flags` を開く
2. "File System Access API" を検索
3. 設定を "Enabled" に変更
4. ブラウザを再起動

⚠️ **注意**: この設定はプライバシーに影響する可能性があります。信頼できるサイトでのみ使用してください。

## 📖 使用方法

### 1. 初回セットアップ（推奨）

1. **HTTPS ページ**を開く（例: https://example.com）
2. 拡張機能アイコンをクリック
3. 「📁 保存先ディレクトリを選択」をクリック
4. 専用フォルダを作成・選択（例: `MarkdownSaves`）

### 2. ページ保存

1. 保存したい Web ページを開く
2. 拡張機能アイコンをクリック
3. 「ページを保存」ボタンをクリック
4. Markdown ファイルが指定ディレクトリに保存される

### 3. ブラウザ別の動作

- **Chrome/Edge**: 完全なディレクトリ選択機能
- **Brave**: フラグ有効化後に利用可能
- **Firefox**: ダウンロードフォルダへの保存
- **HTTP/非対応ブラウザ**: ダウンロードフォルダへのフォールバック

```
│   ├── icon-16.png

├── libs/                  # 外部ライブラリ
│   └── turndown.min.js    # HTML→Markdown変換
├── debug_implementation.js # 開発・テスト用
└── debug_test.html        # デバッグページ
```

### 🔧 主要技術

- **Manifest V3**: 最新の Chrome Extensions API
- **File System Access API**: 永続的なディレクトリアクセス
- **IndexedDB**: ディレクトリハンドルの永続化
- **Turndown.js**: 高品質な HTML→Markdown 変換
- **Content Scripts**: Web ページコンテンツの安全な抽出
- **Service Worker**: バックグラウンド処理とフォールバック

### 🔄 データフロー

1. **コンテンツ抽出**: `content.js`がページ HTML を取得・クリーニング
2. **Markdown 変換**: Turndown.js で高精度変換
3. **メタデータ付与**: タイトル、URL、保存日時を追加
4. **ファイル保存**:
   - 権限あり: File System API で指定ディレクトリに保存
   - 権限なし: Downloads API でダウンロードフォルダに保存
5. **権限管理**: IndexedDB で権限状態を永続化

## 🐛 トラブルシューティング

### よくある問題

**「このブラウザは File System Access API に対応していません」**

- HTTPS ページで実行してください
- Brave の場合はフラグを有効化してください
- 非対応ブラウザでもダウンロードフォルダ保存は利用可能です

**「ディレクトリアクセス権限が拒否されました」**

- ブラウザ設定で権限を確認してください
- ページを更新して再試行してください
- 拡張機能を再読み込みしてください

**「ページコンテンツの取得に失敗しました」**

- ページが完全に読み込まれるまでお待ちください
- JavaScript が有効になっていることを確認してください
- Content Security Policy で制限されている可能性があります

## 📋 制限事項

### File System Access API

- **HTTPS 必須**: HTTP ページでは利用不可
- **ユーザー操作必須**: 自動的なファイル保存は不可
- **ブラウザ依存**: Chromium ベースのブラウザでのみサポート
- **権限期限**: ブラウザセッション終了時に権限失効の可能性

### 一般的な制限

- **CSP 制限**: 一部サイトで Content Security Policy により動作しない
- **動的コンテンツ**: JavaScript 生成コンテンツは保存されない場合がある
- **認証ページ**: ログイン後のページでは動作しない場合がある
- **SPA 制限**: Single Page Application で一部制限あり
